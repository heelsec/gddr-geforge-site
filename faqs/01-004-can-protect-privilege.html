How can the GeForge root privilege escalation be further mitigated? Enabling an IOMMU (Input-Output Memory Management Unit) on your host system is an effective mitigation. On Linux systems, this can be configured by adding intel_iommu=on (for Intel processors) or amd_iommu=on (for AMD processors) to your kernel boot parameters. While this configuration does not prevent the underlying Rowhammer bit flips within the GPU memory, it restricts the GPU's direct memory access (DMA), effectively blocking GeForge from accessing host memory.
